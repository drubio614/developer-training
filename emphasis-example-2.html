Best Practices for Accessible Emphasis

Using emphasis correctly ensures that all users understand the intended meaning and importance of your content.

Core Principles

Use Semantically: Employ <strong> for strong importance, seriousness, or urgency, and <em> for stress emphasis that changes the meaning of a sentence. This provides essential context for screen reader users.

Prioritize Clear Language: For information that is truly critical, ensure the text itself conveys this importance. For example, prefixing crucial instructions with words like "Important:", "Warning:", or "Note:" can be effective for all users.

Use CSS for Visuals: If the intent is purely visual (e.g., making a product name bold for branding purposes without implying strong importance), use CSS properties like font-weight: bold; or font-style: italic; on a <span> or other appropriate element. This avoids conveying unintended semantic meaning.

Avoid Overuse: Excessive use of emphasis can make text harder to read and can diminish the impact of genuinely important or stressed phrases. Use it sparingly to maintain its effectiveness.

Examples

Good Examples

These examples use HTML tags according to their semantic meaning.

Warning: This product contains peanuts.

She was not the person I was expecting.

Our new QuantumLeap browser is the fastest on the market.

<!-- Correct: Using <strong> for a serious warning -->
<p><strong>Warning:</strong> This product contains peanuts.</p>

<!-- Correct: Using <em> to change the meaning of the sentence -->
<p>She was <em>not</em> the person I was expecting.</p>

<!-- Correct: Using CSS (via a class) for purely visual styling -->
<p>Our new <span class="product-name">QuantumLeap</span> browser is the fastest on the market.</p>

Bad Examples

These examples misuse semantic tags for styling, which can confuse users and assistive technologies.

Our Mission

This paragraph is italicized just to make it look different, which makes it harder to read and provides no semantic value. It's better to use CSS for this kind of styling.

<!-- Incorrect: Using <strong> for a heading. A heading tag like <h3> should be used instead. -->
<p><strong>Our Mission</strong></p>

<!-- Incorrect: Using <em> to style an entire paragraph. This overuses emphasis and makes the text difficult to read. -->
<p><em>This paragraph is italicized just to make it look different, which makes it harder to read and provides no semantic value. It's better to use CSS for this kind of styling.</em></p>
