Proper Use of HTML Landmark Roles

Landmarks help users of assistive technologies understand the layout of a page and navigate to main sections easily.

Using HTML5 Elements (Implicit Landmarks)

Modern HTML5 elements like <header>, <nav>, <main>, and <footer> automatically create landmark roles. Using them correctly is the easiest way to make a page's structure accessible.

<header>
  <h1>Site Title</h1>
</header>

<nav aria-label="Primary Site Navigation">
  <ul><li><a href="/">Home</a></li></ul>
</nav>

<main>
  <p>This is the main content of the page.</p>
</main>

<footer>
  <p>© 2024</p>
</footer>

Explicitly Defining a Landmark with `role`

For elements that don't have an implicit role, you can add one using the role attribute. For instance, you can designate a search area. While this works, it's often better to use the native <search> element or add role="search" directly to the <form> tag.

<div role="search">
  <form id="searchform" action="/search" method="get">
    <label for="search-input">Search this site:</label>
    <input type="search" id="search-input" name="q">
    <button type="submit">Search</button>
  </form>
</div>

Labeling Generic Regions

A <section> element can create a generic landmark, but it's not useful to assistive technology unless it's given a unique, accessible name. The aria-labelledby attribute points to the ID of the section's visible heading, using that heading as its label.

<section aria-labelledby="related-links-heading">
  <h2 id="related-links-heading">Related Links</h2>
  <ul>
    <li><a href="/related1">Related Article 1</a></li>
    <li><a href="/related2">Related Article 2</a></li>
  </ul>
</section>
