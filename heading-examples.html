Proper Use of HTML Heading Tags

Correctly structuring headings is crucial for accessibility, SEO, and document clarity.

Good Example: Logical Order

This structure follows a logical, descending order (h1 → h2 → h3) without skipping any levels. This creates a clear outline of the page that can be easily understood by users and assistive technologies.

Page Title: Exploring Web Accessibility

Understanding WCAG Principles

The Web Content Accessibility Guidelines (WCAG) are organized around four main principles: Perceivable, Operable, Understandable, and Robust. Adhering to these helps make web content more accessible to all.

Perceivable

Information and user interface components must be presentable to users in ways they can perceive. This means providing text alternatives for non-text content like images and captions for videos.

Operable

User interface components and navigation must be operable. For example, all functionality should be available from a keyboard, and users must have enough time to read and use content.

Practical Tips for Accessible Design

Beyond principles, practical steps can make a huge difference. Simple changes in how you write your code and design your layouts can greatly improve the user experience for people with disabilities.

Use Semantic HTML

Using HTML elements for their intended purpose provides meaning and structure. This benefits screen reader users who can navigate by headings and helps search engines better understand your content.

Bad Examples

Bad Example (Skipped Level)

Here, an <h4> directly follows an <h2>, skipping the <h3> level. This breaks the logical structure and can be confusing for screen reader users who navigate by headings.

Page Title: Common Baking Mistakes

Mistake 1: Incorrect Measurements

One of the most frequent sources of baking failures is inaccurate measurement of ingredients, especially dry ones like flour and sugar.

Problem: Sub-point on Flour

Bad Example (Multiple <h1>s)

Using more than one <h1> on a page is generally discouraged. A single <h1> should represent the main title of the entire page. Using multiple <h1>s for section titles dilutes the semantic meaning of the primary page title.

My Recipe Blog

<!-- INCORRECT: These should be h2 -->

Delicious Chocolate Cake

This is my favorite recipe for a rich, moist chocolate cake that is perfect for any occasion, from birthdays to simple weekend treats.

Tips for Perfect Frosting

To avoid grainy frosting, make sure your butter is at room temperature and that you sift your powdered sugar before mixing.

Code:

<h1>My Recipe Blog</h1>
<main>
    <p><span class="comment"><!-- INCORRECT: These should be h2 --></span></p>
    <h1>Delicious Chocolate Cake</h1>
    <p>This is my favorite recipe for a rich, moist chocolate cake that is perfect for any occasion, from birthdays to simple weekend treats.</p>
    <h1>Tips for Perfect Frosting</h1>
    <p>To avoid grainy frosting, make sure your butter is at room temperature and that you sift your powdered sugar before mixing.</p>
</main>

A More Appropriate Structure

This is a better way to structure the recipe blog example. It uses a single <h1> for the page's unique title ("Delicious Chocolate Cake") and <h2> tags for the main sections, creating a clear and correct hierarchy.

My Recipe Blog (Site Name/Logo)

Delicious Chocolate Cake

Ingredients

You will need flour, sugar, cocoa powder, baking soda, eggs, buttermilk, and vegetable oil. A full list with measurements is below.

Instructions

Start by preheating your oven to 350°F (175°C). While it heats, whisk together the dry ingredients in a large bowl. Then, add the wet ingredients and mix until just combined.

Tips for Perfect Frosting

For the smoothest chocolate frosting, use a high-quality cocoa powder and ensure all your ingredients, especially butter and cream cheese, are at room temperature before you begin mixing.
