Semantic HTML5 Elements for Accessibility

Semantic HTML Examples

The primary HTML5 elements for structuring a web page allow developers to define distinct regions such as headers, navigation menus, main content areas, and footers. Using these elements correctly provides a clear roadmap of the page for both browsers and assistive technologies.

Element Reference

<header>

The <header> element represents introductory content for its nearest ancestor sectioning content or sectioning root element. Typically, a page <header> contains a site logo, title, primary navigation, and perhaps a search form.

<nav>

The <nav> element defines a section containing navigation links. This is intended for major blocks of navigation, such as the primary site navigation or a table of contents.

<main>

The <main> element represents the dominant content of the <body>. The content inside should be unique to the document and not repeated across pages. There should only be one <main> element per page.

<article>

The <article> element represents a self-contained composition that is independently distributable or reusable, like a blog post, forum post, or news story.

<section>

The <section> element represents a thematic grouping of content, typically with a heading. It's a generic sectioning element for when other, more specific elements aren't appropriate.

<footer>

The <footer> element represents a footer for its nearest sectioning content. A footer typically contains information like the author, copyright data, or links to related documents.

<aside>

The <aside> element represents content that is only indirectly related to the document's main content, often presented as a sidebar or call-out box.

<body>
  <header>
    <h1>My Tech Blog</h1>
    <nav>
      <ul>
        <li><a href="#">Home</a></li>
        <li><a href="#">Tutorials</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <h2>The Importance of Semantic HTML</h2>
      <p>Semantic HTML is crucial for web accessibility...</p>
      <section>
        <h3>Why it Matters for SEO</h3>
        <p>Search engines use semantic markup...</p>
      </section>
      <footer>
        <p>Posted by Alex Doe</p>
      </footer>
    </article>
  </main>

  <aside>
    <h3>Related Posts</h3>
    <ul>
      <li><a href="#">Understanding ARIA</a></li>
    </ul>
  </aside>

  <footer>
    <p>© 2025 My Tech Blog</p>
  </footer>
</body>

Bad Example: Non-Semantic "div soup"

This common anti-pattern uses non-semantic <div> elements for everything. While it can be styled to look identical to the good example, it provides no structural meaning for assistive technologies or search engines.
